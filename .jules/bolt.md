## 2024-05-24 - [O(N) List Queue Operations]
**Learning:** The use of standard Python `list` structures (`list.pop(0)` and `item in list`) for crawler queues creates O(N) operations. Under heavy crawling loads where thousands of URLs are discovered per page, these operations cause the loop execution time to spike drastically (e.g., from ~4.5s down to ~2.2s for just 10 pages containing 3k links each). Furthermore, evaluating fast O(1) membership checks (`not in set`) before performing relatively expensive string parsing (`urlparse`) dramatically improves throughput.
**Action:** When implementing custom crawling or BFS-style traversal loops, strictly use `collections.deque` combined with a distinct `set` for lookup operations, and always order conditionals to execute O(1) cache/set checks before expensive string operations.
