## 2023-10-25 - Web Crawler Queue String Parsing Bottleneck
**Learning:** In the `recursive_crawl` function, executing `urlparse(url).netloc` for every discovered link before checking if it was already visited caused a massive performance bottleneck, slowing down execution significantly on pages with many links. `urlparse` is an expensive string operation that shouldn't be repeatedly performed for previously seen URLs. Additionally, using an O(N) list for `queue` operations (`pop(0)` and `not in queue`) degrades performance exponentially under heavy load.
**Action:** Always perform fast O(1) set membership checks (`url not in visited` and `url not in queued_urls`) before executing expensive string parsing or validation logic. Use `collections.deque` and an auxiliary `set` to track queue state instead of list operations.
