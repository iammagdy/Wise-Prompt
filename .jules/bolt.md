
## 2024-05-18 - Queue Optimization Pattern in Web Crawlers
**Learning:** In crawler loops with heavy queue load (simulating many deep links per page), O(N) list operations like `queue.pop(0)` and containment checks like `if url not in queue` cause significant execution time spikes (e.g., from milliseconds to ~14.1 seconds for 10,000 links). Additionally, using mocked objects (like `MagicMock`) instead of native Python dicts/strings in benchmark loops dominates the actual execution time, masking the true algorithmic performance of the tested function. Finally, the placement of expensive string parsing (e.g., `urlparse`) before fast O(1) checks significantly bottlenecks performance.
**Action:** When optimizing loop bottlenecks, replace lists with `collections.deque` and auxiliary sets (`queued_urls`) for O(1) pops and lookups. Structure conditionals to perform fast native type/set checks *before* making any function calls or parsing logic. When writing isolated performance benchmarks, always mock large iterables using native Python data types rather than heavy `MagicMock` instances to get an accurate representation of loop execution time.
