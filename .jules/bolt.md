## 2025-02-28 - [O(1) Enqueue Lookups in BeautifulSoup Crawler]
**Learning:** Using `not in queue` (where `queue` is a List) during a loop that processes thousands of parsed links on a single web page causes massive quadratic execution spikes (~9s overhead on 5000 links). Additionally, performing string parsing (`urlparse(url).netloc`) *before* checking if the URL has already been visited or enqueued wastes CPU cycles on duplicate links.
**Action:** When crawling, always maintain an auxiliary `set` for queue membership (e.g., `queued_urls`) to turn O(N) list containment checks into O(1) set lookups, and always structure conditionals to perform fast O(1) set logic (like checking `visited` or `queued_urls`) before executing expensive string manipulation or parsing functions.
