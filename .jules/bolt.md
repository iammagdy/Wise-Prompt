## 2024-05-14 - O(N) List Operations in Recursive Crawlers
**Learning:** In the `recursive_crawl` function, using `queue.pop(0)` and `url not in queue` on a standard list causes severe O(N) performance degradation as the queue grows, causing execution time to spike drastically (e.g., from ~25s to 133s under heavy load). Additionally, running expensive string parsing operations like `urlparse(url).netloc` before fast O(1) containment checks wastes CPU cycles on already visited domains.
**Action:** Replace lists with `collections.deque` for O(1) pops and introduce an auxiliary `set` (e.g., `queued_urls`) for O(1) containment checks when managing large queues. Always perform fast O(1) set membership checks (`url not in visited`) before executing expensive operations like `urlparse`.
